웹 서버의 프록시 파일

원격 서버로부터 데이터를 가져오는 첫 번째 방법은 여러분의 서버에 원격 서버로부터 데이터를 수집하는 파일을 생성하는 것이다. 이후 사이트의 다른 페이지는 이 파일에 데이터를 요청한다. 이를 프록시(proxy)라고 하는데, 그 이유는 다른 페이지의 대행자처럼 동작하기 때문이다.

프록시 파일을 생성하는 것은 서버 측 언어를 사용해야 하기 때문에 이 책의 범위를 벗어난다.

JSONP

JSON 데이터를 로드하는 script 요소를 다른 서버로부터 페이지에 추가하는 방법이다. 이 방법이 가능한 이유는 script 태그의 원본 주소에 대한 제약이 존재하지 않기 때문이다.


스크립트는 함수를 호출하며, 함수의 매개변수로는 JSON 형식의 데이터가 전달된다. 이때 호출되는 함수는 페이지에 정의되는 것이며, 페이지를 처리하고 표시하는 역할을 담당한다.

크로스 오리진 리소스 공유

브라우저는 서버와 통신할 때마다 서로에 대한 정보를 HTTP 헤더를 이용해 전달한다. 크로스 오리진 리소스 공유(CORS: Cross-Origin Resource Sharing)는 HTTP 헤더에 추가적으로 정보를 추가하여 브라우저와 서버가 서로 통신을 해야 한다는 사실을 알게 한다.
CORS는 W3C 명세에 포함되어 있지만 최신 버전의 브라우저에서만 동작하며 서버에 HTTP 헤더를 설정해야 하기 때문에 역시 이 책의 범위를 벗어난다.
