THE MAKER BUS

라이브 검색

라이브 검색(혹은 자동 완성)이라고 불리는 기능은 주로 Ajax를 이용한다. 아마 구글 웹사이트를 사용할 때 한 번쯤 경험했을 것이다. 검색 막대나 홈페이지에서 검색어를 입력하는 동시에 검색 결과가 나타나는 것을 볼 수 있을 것이다.

사용자 정보 표시

사용자가 생성한 콘텐츠를 사용하는 웹사이트(예를 들면, 트위터나 플리커 등)는 독자들의 웹사이트에 본인의 정보(최근 트윗이나 사진 등)를 노출하는 기능을 제공한다. 이를 위해 자신들의 서버에 데이터를 수집한다.

장바구니

간혹 온라인에서 쇼핑을 하면서 장바구니에 아이템을 추가하면 해당 변경 사항이 페이지를 이동하지 않고도 이루어진다. 그와 동시에 사이트는 아이템을 추가했음을 알리는 메시지를 보여주기도 한다.

회원가입

웹사이트에 회원가입을 할 때 스크립트가 나머지 폼을 다 입력하기 전에 사용자 이름의 사용 가능 여부를 알려주기도 한다.

Ajax란 무엇인가

Ajax는 전체 페이지를 새로 고치지 않고도 페이지의 일부만을 위한 데이터를 로드하는 기법이다. 데이터는 주로 JSON(Javascript Object Notation, 자바스크립트 객체 식)이라고 부르는 형태로 전달된다.

Ajax를 사용하는 이유

사용자는 웹 브라우저가 데이터를 로드하는 동안 기다리지 않고 다른 일을 할 수 있으므로 더욱 빠른 사용자 경험을 제공할 수 있다.

로드되는 동안 사용하기

페이지가 로드되는 동안 스크립트가 서버에서 데이터를 수집해야 하면, 브라우저는 스크립트가 서버로부터 데이터를 수집하고 이를 처리한 후 페이지의 나머지 부분이 모두 로드될 때까지 대기한다.

로드된 후에 사용하기

일단 페이지가 로드된 후에 사용자가 브라우저 창을 통해 보고 있는 것을 수정해야 한다면, 일반적으로 전체 페이지를 새로 고치게 된다. Ajax를 이용하면 페이지의 일부를 수정하고 싶을 때 어느 특정 요소의 컨텐츠를 업데이트 하면 된다. 그러려면 이벤트를 가로채서 서버에 새로운 콘텐츠를 요청하는 비동기 요청을 보내면 된다.

요청

브라우저는 서버에 정보를 요청한다. 이 요청은 서버가 필요로 하는 정보를 포함하고 있을 수도 있다. 브라우저는 Ajax 요청을 담당하는 XMLHttpRequest라는 객체를 구현하고 있다. 일단 요청을 전송하고 나면, 브라우저는 서버의 응답을 기다리지 않는다.

서버의 동작

서버에서 일어나는 일들은 Ajax라고 부르는 처리 과정에 포함되지 않는다. ASP.NET, PHP, NodeJS, 루비 같은 서버 측 기술들은 사용자마다 다른 웹 페이지를 만들어낼 수 있다. 서버는 Ajax 요청을 전달받으면 HTML이나 JSON 또는 XML 형식의 데이터를 전달한다.

응답

서버가 요청에 대한 응답을 완료하면 브라우저에서는 이벤트가 발생한다. 이 이벤트는 데이터를 처리하여 (페이지 내의 다른 영역에는 아무런 영향을 주지 않으면서) 페이지의 일부로 변환하는 자바스크립트 함수를 호출한다.
